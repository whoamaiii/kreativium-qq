## Brief overview
These guidelines outline preferences for UI component libraries, specifically regarding the use of Radix UI.

## UI Component Preferences
- **Radix UI Primitives:** Default to using Radix UI primitives when implementing UI elements such as dialogs, dropdowns, or popovers. This ensures consistency and leverages the accessibility and customization benefits of Radix UI.
  - *Trigger:* When a task involves adding or modifying dialogs, dropdown menus, popover components, or similar interactive UI elements.
  - *Example:* Instead of building a custom dropdown, use Radix UI's DropdownMenu primitive.
